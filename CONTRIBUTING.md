# Contributing Guide

## Комміти
- Conventional Commits: `feat:`, `fix:`, `docs:`, `chore:`…

## Гілки
- `main` — стабільна. Розробка у feature-гілках → PR.

## Стиль коду
- Node 20+, ESM/TypeScript (пізніше). Поки що — прості скрипти на Node.
- ESLint/Prettier (додамо в першому спринті).

## Як взаємодіяти з ІІ-розробником
- Описуйте задачу в issue (шаблон `feature.yml` / `bug.yml`).
- Асистент читає `docs/*` і відповідає у PR/логах CI.
